# Задание 3. Архитектура PUSH-уведомлений

В мобильном приложении интернет-магазина PUSH-уведомления отправляются
в виде реакции на события, происходящие в бизнес-сервисах
(корзина, заказы, маркетинг)

Бизнес-сервисы публикуют события в очередь событий и не содержат логики
отправки уведомлений. Решение о необходимости отправки PUSH,
его содержании и времени отправки принимает отдельный сервис уведомлений

Для доставки PUSH-уведомлений используется внешний push-сервис
(APNS / FCM / HMS или агрегатор), который доставляет уведомления
в мобильное приложение пользователя
